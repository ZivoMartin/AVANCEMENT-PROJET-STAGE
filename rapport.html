The recorder/replayer software

A recorder/replayer is a software that first records a sequence of actions performed when interacting with the browser, then converts this sequence of actions into a script.
Once executed, the script will call the replayer, which will read the script and execute the actions one by one on the browser. This testing software allows one to quickly and automatically test a web application rather than having to test all the application's functions each time it is modified. All you have to do is run the script, which is very convenient.

However, recorders do have their weaknesses. The recorder listens to JavaScript events and records them.
However, some events are not detected by the recorder, which leads to problems during replay. The aim of this report is therefore to compare four test programs: Cypress, Selenium, Playwright and Chrome, and to determine the strengths and weaknesses of each.

In this study, we first selected JavaScript events divided into two categories: those in the "mouse events" class and those in the "keyboard events" class. This table compares the management of keyboard and mouse events for each software. It does not deal with sequences of events (scenarios), but rather with single events.

To carry out the tests, I decide to use 3 different sites for greater precision, so all 3 must have the necessary functionality to test all the criteria. So we have to find 3 sites that have a hover menu, a combo box, and a search text field. Most of the other criteria are found on all websites. It's also much more efficient to test on the most popular sites, so I pick the top 3 sites with all the features from the official rankings (https://www.similarweb.com/top-websites/) and test on them. So, the first one is eBay (https://www.ebay.com/), the second is Samsung (https://www.samsung.com/fr/), and the last one is Wikipedia (https://fr.wikipedia.org/wiki/Wikip%C3%A9dia:Accueil_principal).

The CSS selector algorithm

So far, we have compared software on the basis of its ability to perform actions.
This study will go further. To retrieve an element from the HTML DOM of the web page, the software uses a CSS selector algorithm to retrieve the element. This is a complicated subject that raises a lot of questions.
This study will compare the different selection algorithms of the three software packages. To do this, we will visit 10 sites from the top 10 most popular sites and make 5 clicks. We will then assign a score to each software package according to the number of clicks accepted. The 10 sites in question will be : Google, Amazon, Openai, Microsoft, Globo, Wikipedia, Ebay, Dzen, Office, Samsung. For the selection of the different sites, we simply did not take the sites that require a connection like instagram, facebook... Because in general, the replayer is not accepted by the site.
